Images: The Blueprint

An image is a stack of layers

Not a single file — a stack of read-only, immutable layers. Each represents a set of filesystem changes. When a container runs, Docker adds one thin writable layer on top, unique to that container.

Union filesystem

Merges every layer into one unified view. Like stacked transparent glass sheets — each sheet independent, but together they form one complete picture.

Copy-on-write (CoW)

When a container modifies a file that lives in a read-only image layer, Docker copies that file up to the container's writable layer first, then modifies the copy. The original in the image is never touched — which is why five containers from the same image never interfere with each other.

verified live Wrote a file inside one container's /tmp. A second container from the same image couldn't see it — proof the file only existed in the first container's own writable layer, not in the shared image.

Why layers matter: caching

Two images that both start from Ubuntu and install the same libraries don't store that data twice — Docker recognizes identical layers and stores them once, shared across images. This is also why pulling a second Python image is faster than the first: many base layers are already local.

Tags are pointers, not copies

docker tag hello-world:latest anshul/my-first-image:v1.0
docker images   # same Image ID, two different tag entries

Tagging never duplicates data — it just creates a new label pointing at the same underlying image ID.

production rule Never use :latest in production. It's not "auto-updating" — it's just "whatever was tagged latest at pull time," which breaks reproducibility. Pin exact versions: python:3.11.4-slim, not python:latest.

Base image sizes — measured, not assumed

ImageSize
python:3.111.6 GB
python:3.11-slim188 MB
python:3.11-alpine84 MB

Alpine is smallest but uses musl libc instead of glibc, which can break Python packages with C extensions. slim is usually the safer default for Python apps — most of the size win, none of the compatibility risk.

Useful commands

docker history nginx:latest       # see every layer + size
docker inspect nginx:latest       # full JSON detail
docker system df                  # disk usage summary
docker system prune -a            # remove ALL unused images/containers
